# 자바 기초 다지기

자바 개발환경(플랫폼)을 구축할 때 다른 플랫폼을 쓰는 이유: 목적이 다르기 때문에

- 데스크톱 응용소프트웨어: JavaSE
- 웹 기반: JavaEE
- 모바일 기반: JavaME

개발환경 구축 → JavaSE플랫폼 구축(JDK설치)

예) 문제 제시 

코드를 짠다 → 구현

저장한다: [Calculator.java](http://Calculator.java)

문법오류 체크: 컴파일러(해석기)필요(javac.exe 파일으로 컴파일) → JDK

만들어지는 파일: Calculator.class(바이트 코드 파일)

실행한다: 실행해주는 명령어 및 프로세서 필요(java.exe, JVM) → JDK[JRE(JVM)]

JDK를 설치 했다는 것은… JVM, API, TOOL(명령어)가 PC에 설치 되었다는 뜻.

- 자바 API는 자바에서 제공해주는 클래스들의 집합이다.
- 자바 가상 머신(JVM)은 자바 프로그램을 실행해주는 프로세서이다 (엔진)
- TOOL은 자바소스 코드를 컴파일하거나 실행하는 등에 필요한 명령어들의 집합이다.

JDK를 설치하면 JVM이 메모리 위에 탑재된다. 어떤 한 프로그램이 실행되기 위해서는 메모리 상에 로드하고 실행시켜야 하는데 이 일을 JVM이 해준다. 따라서 java.exe 파일은 프로그램을 직접적으로 실행시키는 파일이 아니라 JVM을 호출 해주는 파일인 것이다.

JDK를 설치하면 들어있는 파일들 중에 bin에는 TOOL이 들어있고 lib디렉토리에는 API가 들어있다.

jvm은 표면적으로 안보이나 메모리상에서 동작한다.
